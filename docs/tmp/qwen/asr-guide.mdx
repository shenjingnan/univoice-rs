
# QwenASR 使用指南

本文介绍如何使用 univoice SDK 中的通义千问 ASR（语音识别）功能。

## 概述

QwenASR 基于阿里云 DashScope Paraformer WebSocket API 实现语音识别，使用 **DashScope 双向流式 WebSocket 协议**，支持边发边收的实时识别。

| 特性 | 说明 |
|------|------|
| 协议 | DashScope WebSocket 双向流式协议 |
| 端点 | `wss://dashscope.aliyuncs.com/api-ws/v1/inference/` |
| 认证 | Bearer Token（API Key） |
| 音频格式 | mp3、wav、pcm（原生支持，无需转换） |
| 流式识别 | 支持双向流式（边发边收） |
| 非流式识别 | 支持一次性返回完整结果 |

支持三个 Paraformer 模型：

| 模型 | 说明 | 适用场景 |
|------|------|----------|
| `paraformer-realtime-v2`（推荐） | 支持多语言、任意采样率 | 通用场景 |
| `paraformer-realtime-v1` | 支持 16kHz 采样率 | 标准语音识别 |
| `paraformer-realtime-8k-v1` | 支持 8kHz 采样率 | 电话语音、低采样率场景 |

## 创建 ASR 实例

univoice 提供两种创建 QwenASR 实例的方式：

<Tabs items={['工厂函数（推荐）', '直接实例化']}>
<Tab>

使用 `createASR` 工厂函数，需先导入 `univoice/asr/providers` 注册提供商：

```typescript
import 'univoice/asr/providers';
import { createASR } from 'univoice/asr';

const asr = createASR({
  provider: 'qwen',
  apiKey: process.env.QWEN_API_KEY,
  model: 'paraformer-realtime-v2',
});
```

</Tab>
<Tab>

直接导入 `QwenASR` 类并实例化，无需注册 provider：

```typescript
import { QwenASR } from 'univoice/asr';

const asr = new QwenASR({
  apiKey: process.env.QWEN_API_KEY,
  model: 'paraformer-realtime-v2',
});
```

</Tab>
</Tabs>

<Callout type="info">
两种方式创建的实例拥有完全一致的 API，推荐使用工厂函数，便于切换不同提供商。
</Callout>

## 非流式识别

最简单的使用方式 —— 传入音频文件路径，一次性获取完整识别结果：

```typescript
import 'univoice/asr/providers';
import { createASR } from 'univoice/asr';

const asr = createASR({
  provider: 'qwen',
  apiKey: process.env.QWEN_API_KEY,
  model: 'paraformer-realtime-v2',
});

// 传入文件路径，返回完整识别结果
const result = await asr.listen('/path/to/audio.mp3');

console.log('识别结果:', result.text);

// 查看分段信息
if (result.segments) {
  for (const segment of result.segments) {
    console.log(`[${segment.start}ms - ${segment.end}ms] ${segment.text}`);
  }
}
```

### ASRResponse 结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `text` | `string` | 完整识别文本 |
| `language` | `string?` | 检测到的语言 |
| `duration` | `number?` | 音频时长（ms） |
| `segments` | `ASRSegment[]?` | 分段信息 |

## 流式识别

流式识别适合实时场景，音频边发送边接收识别结果：

```typescript
import 'univoice/asr/providers';
import { createASR } from 'univoice/asr';
import { bufferToAudioStream } from 'univoice/asr';

const asr = createASR({
  provider: 'qwen',
  apiKey: process.env.QWEN_API_KEY,
  model: 'paraformer-realtime-v2',
});

// 将 Buffer 转为音频流
const audioStream = bufferToAudioStream(audioBuffer);

// 流式识别 - 边发边收
for await (const chunk of asr.listen(audioStream, { stream: true })) {
  const status = chunk.isFinal ? '最终' : '中间';
  console.log(`[${status}] ${chunk.text}`);
}
```

### ASRStreamChunk 结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `text` | `string` | 本次识别的文本片段 |
| `isFinal` | `boolean` | 是否为最终结果（中间结果可能被修正） |
| `confidence` | `number?` | 置信度 |
| `segment` | `ASRSegment?` | 分段信息 |

<Callout type="info">
流式识别会持续返回中间结果（`isFinal: false`），每个最终结果（`isFinal: true`）代表一个稳定的识别片段。通常需要将所有最终结果拼接起来得到完整文本。
</Callout>

## 模型选择

QwenASR 支持三个 Paraformer 模型，根据不同场景选择：

| 模型 | 采样率 | 特点 | 推荐场景 |
|------|--------|------|----------|
| `paraformer-realtime-v2` | 任意 | 多语言、高性能 | **通用场景（推荐）** |
| `paraformer-realtime-v1` | 16kHz | 标准识别 | 标准语音识别 |
| `paraformer-realtime-8k-v1` | 8kHz | 电话语音优化 | 电话录音、低采样率场景 |

```typescript
// 推荐：v2 模型，支持多语言和任意采样率
const asr = createASR({
  provider: 'qwen',
  apiKey: process.env.QWEN_API_KEY,
  model: 'paraformer-realtime-v2',
});

// 电话语音场景：8k-v1 模型
const asr8k = createASR({
  provider: 'qwen',
  apiKey: process.env.QWEN_API_KEY,
  model: 'paraformer-realtime-8k-v1',
});
```

## 音频格式支持

QwenASR 原生支持多种音频格式，SDK 会自动检测采样率，**无需预先转换为 PCM**：

<Tabs items={['MP3', 'WAV', 'PCM']}>
<Tab>

MP3 格式可以直接使用，SDK 会自动检测参数并发送：

```typescript
const asr = createASR({
  provider: 'qwen',
  apiKey,
  model: 'paraformer-realtime-v2',
});

// 直接传入 MP3 文件路径，SDK 自动处理
const result = await asr.listen('/path/to/audio.mp3');
```

</Tab>
<Tab>

WAV 格式自带音频参数头信息，SDK 会自动解析采样率等参数：

```typescript
const asr = createASR({
  provider: 'qwen',
  apiKey,
  model: 'paraformer-realtime-v2',
});

// 直接传入 WAV 文件路径
const result = await asr.listen('/path/to/audio.wav');
```

</Tab>
<Tab>

PCM 格式需要指定采样率，或通过 `audioFormat` 配置：

```typescript
const asr = createASR({
  provider: 'qwen',
  apiKey,
  model: 'paraformer-realtime-v2',
  format: 'pcm',
  audioFormat: {
    sampleRate: 16000,
  },
});
```

</Tab>
</Tabs>

<Callout type="info">
与豆包 ASR 不同，Qwen ASR 原生支持 mp3/wav/pcm 格式。传入文件路径时，SDK 会自动检测采样率，无需手动转换为 PCM。
</Callout>

## Opus 音频处理

在实时语音场景中，音频通常以 Opus 编码传输。使用 `decodeOpusStream` 将 Opus 数据包解码为 PCM 后流式发送：

```typescript
import 'univoice/asr/providers';
import { createASR, decodeOpusStream } from 'univoice/asr';

const asr = createASR({
  provider: 'qwen',
  apiKey: process.env.QWEN_API_KEY,
  model: 'paraformer-realtime-v2',
  audioFormat: {
    sampleRate: 16000,
  },
});

// 将 Opus 数据包解码为 PCM 流
const audioStream = decodeOpusStream(opusPacketStream, {
  sampleRate: 16000,
});

for await (const chunk of asr.listen(audioStream, { stream: true })) {
  console.log(chunk.text);
}
```

## 完整配置参考

### QwenASROptions

#### 认证配置

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `apiKey` | `string` | `''` | 阿里云 DashScope API Key（**必填**） |

#### 识别配置

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `model` | `string` | `'paraformer-realtime-v2'` | Paraformer 模型名称 |
| `language` | `string?` | `undefined` | 识别语言（如 `'zh-CN'`、`'en-US'`），SDK 自动转换为 `language_hints` |
| `enableWords` | `boolean?` | `undefined` | 是否启用词级时间戳 |
| `enablePunc` | `boolean?` | `undefined` | 是否启用标点预测 |
| `enableItn` | `boolean?` | `undefined` | 是否启用逆文本标准化（数字转阿拉伯数字等） |

#### 音频配置

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `format` | `'pcm' \| 'wav' \| 'ogg' \| 'mp3'` | `'mp3'` | 音频容器格式 |
| `audioFormat.sampleRate` | `number?` | 自动检测 | 音频采样率 |

#### 通用配置

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `baseUrl` | `string` | `'wss://dashscope.aliyuncs.com/api-ws/v1/inference/'` | WebSocket 服务地址 |
| `prompt` | `string?` | `undefined` | 提示文本 |
| `responseFormat` | `'json' \| 'text' \| 'srt' \| 'vtt' \| 'verbose_json'` | `'json'` | 响应格式 |

## 工具函数

### collectText

收集识别结果文本，支持分段回调：

```typescript
import { collectText } from 'univoice/asr';

const result = await asr.listen('/path/to/audio.mp3');
const text = await collectText(result, {
  onSegment: (segment) => console.log('分段:', segment),
  onComplete: (text) => console.log('完成:', text),
});
```

### saveText

将识别结果保存到文件，支持多种格式：

```typescript
import { saveText } from 'univoice/asr';

const result = await asr.listen('/path/to/audio.mp3');

// 保存为纯文本
await saveText(result, { filename: 'result.txt' });

// 保存为 JSON（包含完整响应结构）
await saveText(result, { format: 'json', directory: './output' });
```

### bufferToAudioStream

将 Buffer 转为音频流，用于流式识别的输入：

```typescript
import { bufferToAudioStream } from 'univoice/asr';

const audioBuffer = await fs.readFile('/path/to/audio.pcm');
const stream = bufferToAudioStream(audioBuffer);

for await (const chunk of asr.listen(stream, { stream: true })) {
  console.log(chunk.text);
}
```

## 常见问题

### 安装 prism-media 后运行报错

**症状**：使用 `decodeOpusStream` 时抛出模块未找到错误。

**原因**：`prism-media` 是可选依赖，需要手动安装。

**解决**：
```bash
pnpm add prism-media
```

同时确保系统已安装 `ffmpeg`：
```bash
# macOS
brew install ffmpeg

# Ubuntu
sudo apt install ffmpeg
```

### ffmpeg 缺失导致音频转换失败

**症状**：使用 `decodeOpusStream` 时抛出 `ffmpeg not found` 错误。

**解决**：安装 `ffmpeg` 并确保在 PATH 中可用：
```bash
ffmpeg -version  # 验证安装
```

### 应该使用文件路径还是音频流作为输入？

- **文件路径**：适合非流式场景，SDK 自动处理文件读取和采样率检测
- **音频流（AudioStream）**：适合流式场景，如实时麦克风输入或网络音频流
- **Buffer**：适合已在内存中的音频数据

三种方式都可以传入 `asr.listen()` 方法，SDK 会自动适配。

### 流式和非流式模式如何选择？

- **非流式**（默认）：传入完整音频，一次性获取结果。适合文件识别、离线处理
- **流式**（`{ stream: true }`）：边发送边接收，实时获取中间结果。适合实时对话、语音输入

### 如何获取 DashScope API Key？

1. 前往 [阿里云 DashScope 控制台](https://dashscope.console.aliyun.com/)
2. 开通「语音识别」服务
3. 在「API Keys」管理页面创建新的 API Key

### Qwen ASR 是否支持多语言识别？

`paraformer-realtime-v2` 模型支持多语言识别。通过 `language` 参数指定语言（如 `'zh-CN'`、`'en-US'`），SDK 会自动将其转换为 DashScope 所需的 `language_hints` 格式。

### enableItn、enablePunc、enableWords 有什么作用？

- **`enableItn`**（逆文本标准化）：将口语化的数字转为阿拉伯数字，如 "一百二十三" → "123"
- **`enablePunc`**（标点预测）：自动在识别结果中添加标点符号
- **`enableWords`**（词级时间戳）：返回每个词的精确起止时间
