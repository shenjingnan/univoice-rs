title: 通义千问 CosyVoice TTS 使用指南
---

本文介绍如何使用 univoice SDK 中的通义千问 TTS（文字转语音）功能。

## 概述

QwenTTS 基于阿里云 DashScope CosyVoice WebSocket API 实现语音合成，提供 **两个 Provider**：

| Provider | 类名 | 端点 | 说明 |
|----------|------|------|------|
| `qwen` | `QwenTTS` | CosyVoice API | CosyVoice 系列模型，音色丰富 |
| `qwen-realtime` | `QwenRealtimeTTS` | Realtime API | Realtime 系列模型，支持 instructions 情感控制 |

两个 Provider 都支持非流式和流式两种合成模式：

| 模式 | 方法 | 说明 | 适用场景 |
|------|------|------|----------|
| 非流式 | `synthesize` | 一次性返回完整音频 | 文件合成、离线处理 |
| 流式 | `speak` + `stream: true` | 边合成边接收音频块 | 实时播报、LLM 流式输出转语音 |

### Provider 选择建议

- **`qwen`（CosyVoice）**：音色丰富（80+ 内置音色），推荐大多数场景使用
- **`qwen-realtime`（Realtime）**：支持 `instructions` 指令控制情感语气，适合需要精细控制语音表现的场景

## 创建 TTS 实例

### CosyVoice 模型（qwen provider）

<Tabs items={['工厂函数（推荐）', '直接实例化']}>
<Tab>

使用 `createTTS` 工厂函数，需先导入 `univoice/tts/providers` 注册提供商：

```typescript
import 'univoice/tts/providers';
import { createTTS } from 'univoice/tts';

const tts = createTTS({
  provider: 'qwen',
  apiKey: process.env.QWEN_API_KEY,
  model: 'cosyvoice-v3-flash',
  voice: 'longxiaochun_v3',
  format: 'mp3',
});
```

</Tab>
<Tab>

直接导入 `QwenTTS` 类并实例化，无需注册 provider：

```typescript
import { QwenTTS } from 'univoice/tts/providers';

const tts = new QwenTTS({
  apiKey: process.env.QWEN_API_KEY,
  model: 'cosyvoice-v3-flash',
  voice: 'longxiaochun_v3',
  format: 'mp3',
});
```

</Tab>
</Tabs>

### Realtime 模型（qwen-realtime provider）

<Tabs items={['工厂函数（推荐）', '直接实例化']}>
<Tab>

使用 `createTTS` 工厂函数，指定 `qwen-realtime` provider：

```typescript
import 'univoice/tts/providers';
import { createTTS } from 'univoice/tts';

const tts = createTTS({
  provider: 'qwen-realtime',
  apiKey: process.env.QWEN_API_KEY,
  model: 'qwen3-tts-instruct-flash-realtime',
  voice: 'Cherry',
  format: 'pcm',
  realtime: {
    instructions: '请用温柔、亲切的语气说话',
  },
});
```

</Tab>
<Tab>

直接导入 `QwenRealtimeTTS` 类并实例化：

```typescript
import { QwenRealtimeTTS } from 'univoice/tts/providers';

const tts = new QwenRealtimeTTS({
  apiKey: process.env.QWEN_API_KEY,
```

## 非流式合成

最简单的使用方式 —— 传入文本，一次性获取完整音频：

```typescript
import 'univoice/tts/providers';
import { createTTS } from 'univoice/tts';

const tts = createTTS({
  provider: 'qwen',
  apiKey: process.env.QWEN_API_KEY,
  model: 'cosyvoice-v3-flash',
  voice: 'longxiaochun_v3',
  format: 'mp3',
});

// 传入文本，返回完整音频
const response = await tts.synthesize({
  text: '欢迎来到杭州！我是您的智能导游。',
});

console.log('音频大小:', response.audio.length, 'bytes');
console.log('音频格式:', response.format);
```

### TTSResponse 结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `audio` | `Buffer \| Uint8Array` | 音频数据 |
| `format` | `string` | 音频格式（如 `mp3`、`pcm`） |
| `duration` | `number?` | 音频时长 |

## 流式合成

流式合成适合实时场景，文本边发送边接收音频块，也可接收 LLM 流式文本输出：

### 字符串输入

```typescript
import 'univoice/tts/providers';
import { createTTS } from 'univoice/tts';

const tts = createTTS({
  provider: 'qwen',
  apiKey: process.env.QWEN_API_KEY,
  model: 'cosyvoice-v3-flash',
  voice: 'longxiaochun_v3',
  format: 'mp3',
});

// 传入字符串，流式接收音频块
for await (const { audioChunk } of tts.speak('欢迎来到杭州！', { stream: true })) {
  console.log('收到音频块:', audioChunk.length, 'bytes');
}
```

### 文本流输入（LLM 流式输出转语音）

```typescript
import 'univoice/tts/providers';
import { createTTS } from 'univoice/tts';

const tts = createTTS({
  provider: 'qwen',
  apiKey: process.env.QWEN_API_KEY,
  model: 'cosyvoice-v3-flash',
  voice: 'longxiaochun_v3',
  format: 'mp3',
});

// 模拟 LLM 流式输出
async function* textStream() {
  yield '欢迎来到杭州！';
  yield '我是您的智能导游。';
}

// 流式文本输入 → 流式音频输出
for await (const { audioChunk } of tts.speak(textStream(), { stream: true })) {
  console.log('收到音频块:', audioChunk.length, 'bytes');
}
```

### TTSStreamChunk 结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `audioChunk` | `Uint8Array` | 音频数据块 |

## 模型版本对比

### CosyVoice 模型（qwen provider）

| 特性 | `cosyvoice-v1` | `cosyvoice-v2` | `cosyvoice-v3-flash` | `cosyvoice-v3-plus` |
|------|----------------|----------------|----------------------|---------------------|
| 默认状态 | — | — | **默认使用** | — |
| 速度 | 标准 | 标准 | 快 | 标准 |
| 质量 | 标准 | 较高 | 较高 | **最高** |
| 成本 | 标准 | 标准 | 低 | 较高 |
| 音色数量 | 20 | 100+ | 60+ | 2 |
| 方言支持 | 不支持 | 支持 | 支持 | 支持 |
| 推荐场景 | 兼容旧版 | 通用场景 | **通用场景（推荐）** | 高质量场景 |

### Realtime 模型（qwen-realtime provider）

| 模型 | 说明 | 特点 |
|------|------|------|
| `qwen3-tts-instruct-flash-realtime`（默认） | 支持 instructions 情感控制 | 可控制语气、情感等 |
| `qwen3-tts-flash-realtime` | 基础 Realtime 模型 | 标准 Realtime 合成 |

通过 `model` 参数切换模型版本：

```typescript
// CosyVoice：使用 v3-plus 高质量模型
const tts1 = createTTS({
  provider: 'qwen',
  apiKey,
  model: 'cosyvoice-v3-plus',
});

// Realtime：使用基础 flash 模型
const tts2 = createTTS({
  provider: 'qwen-realtime',
  apiKey,
  model: 'qwen3-tts-flash-realtime',
});
```

## instruction 情感控制

CosyVoice 和 Realtime 模型都支持通过指令控制语音的情感和风格。

### CosyVoice 的 instruction 参数

通过 `instruction` 配置项控制情感：

```typescript
const tts = createTTS({
  provider: 'qwen',
  apiKey,
  model: 'cosyvoice-v3-flash',
  voice: 'longxiaochun_v3',
  instruction: '请用温柔的语调朗读',
});
```

### Realtime 的 instructions 参数

通过 `realtime.instructions` 配置项控制情感，**仅 `qwen3-tts-instruct-flash-realtime` 模型支持**：

```typescript
const tts = createTTS({
  provider: 'qwen-realtime',
  apiKey,
  model: 'qwen3-tts-instruct-flash-realtime',
  voice: 'Cherry',
  format: 'pcm',
  realtime: {
    instructions: '请用温柔、亲切的语气说话',
  },
});
```

## 音频格式支持

两个 Provider 支持以下音频格式，通过 `format` 参数指定：

<Tabs items={['MP3', 'PCM', 'WAV', 'Opus', 'FLAC / OGG']}>
<Tab>

MP3 是最常用的音频格式，兼容性好，文件体积小：

```typescript
const tts = createTTS({
  provider: 'qwen',
  apiKey,
  format: 'mp3',
});

const response = await tts.synthesize({ text: '你好世界' });
// response.audio 为 MP3 数据，可直接保存为 .mp3 文件
```

</Tab>
<Tab>

PCM 是原始音频数据，无压缩、无容器封装，延迟最低：

```typescript
const tts = createTTS({
  provider: 'qwen',
  apiKey,
  format: 'pcm',
  sampleRate: 24000,
});

const response = await tts.synthesize({ text: '你好世界' });
// response.audio 为 PCM 原始数据
```

<Callout type="info">
PCM 格式不包含文件头信息，播放时需要指定采样率、位深和声道数。默认配置为 24000Hz、16bit、单声道。
</Callout>

</Tab>
<Tab>

WAV 格式包含文件头，可直接播放：

```typescript
const tts = createTTS({
  provider: 'qwen',
  apiKey,
  format: 'wav',
  sampleRate: 24000,
});
```

</Tab>
<Tab>

Opus 格式压缩率高，适合流式传输场景。输出的 Opus 数据使用 OGG 容器封装：

```typescript
const tts = createTTS({
  provider: 'qwen',
  apiKey,
  format: 'opus',
});

const response = await tts.synthesize({ text: '你好世界' });
// 输出的 Opus 音频，保存为 .ogg 文件
```

</Tab>
<Tab>

FLAC 无损压缩格式和 OGG 格式：

```typescript
// FLAC 无损格式
const tts1 = createTTS({
  provider: 'qwen',
  apiKey,
  format: 'flac',
});

// OGG 格式
const tts2 = createTTS({
  provider: 'qwen',
  apiKey,
  format: 'ogg',
});
```

</Tab>
</Tabs>

## 工具函数

### collectAudio

收集音频数据，支持回调：

```typescript
import { collectAudio } from 'univoice/tts';

const response = await tts.synthesize({ text: '你好世界' });
const audio = await collectAudio(response, {
  onComplete: (audio) => console.log('音频大小:', audio.length),
});
```

### saveTTSResponse

将 TTS 响应保存到文件：

```typescript
import { saveTTSResponse } from 'univoice/tts';

const response = await tts.synthesize({ text: '你好世界' });

// 自动生成文件名
const filepath = await saveTTSResponse(response);

// 指定文件名和目录
await saveTTSResponse(response, {
  filename: 'output.mp3',
  directory: './audio',
});
```

### saveAudio

将音频块保存到文件，支持自动添加 WAV 头：

```typescript
import { saveAudio } from 'univoice/tts';

// 保存流式音频块
const stream = tts.speak('你好世界', { stream: true });
await saveAudio('output.wav', stream, { sampleRate: 24000 });
```

### playAudio

播放音频（需要系统安装播放器，macOS 默认使用 `afplay`）：

```typescript
import { playAudio } from 'univoice/tts';

const response = await tts.synthesize({ text: '你好世界' });
await playAudio(response);

// 指定播放器
await playAudio(response, { player: 'ffplay' });
```

### teeAudio

同时保存和播放音频：

```typescript
import { teeAudio } from 'univoice/tts';

const response = await tts.synthesize({ text: '你好世界' });
await teeAudio(response, {
  save: { filename: 'output.mp3', directory: './audio' },
  play: { player: 'afplay' },
});
```

## 常见问题

### 非流式和流式模式如何选择？

- **非流式**（`synthesize`）：传入完整文本，一次性获取音频。适合文件合成、离线处理
- **流式**（`speak` + `stream: true`）：边发送边接收，实时获取音频块。适合实时播报、LLM 流式输出转语音

### CosyVoice 和 Realtime 模型如何选择？

- **CosyVoice**（`qwen` provider）：音色丰富（80+），支持多种格式，适合一般场景
- **Realtime**（`qwen-realtime` provider）：支持 `instructions` 情感控制，适合需要精细控制语音表现的场景

### 如何获取 DashScope API Key？

1. 前往 [阿里云 DashScope 控制台](https://dashscope.console.aliyun.com/)
2. 开通「语音合成」服务
3. 在「API Keys」管理页面创建新的 API Key

### 如何选择 CosyVoice 模型版本？

- **`cosyvoice-v3-flash`**（默认，推荐）：速度快、成本低，适合大多数场景
- **`cosyvoice-v3-plus`**：最高质量，音色较少，适合对音质要求极高的场景
- **`cosyvoice-v2`**：功能丰富，音色最多（100+），适合需要特定音色的场景
- **`cosyvoice-v1`**：仅用于兼容旧版系统

### PCM 格式音频如何播放？

PCM 是原始音频数据，需要指定参数播放：

```bash
ffplay -autoexit -f s16le -ar 24000 -ac 1 output.pcm
```

或者使用 `saveAudio` 保存为 WAV 格式（自动添加文件头）：

```typescript
import { saveAudio } from 'univoice/tts';

const response = await tts.synthesize({ text: '你好世界' });
// 保存为 WAV 文件，自动添加文件头
await saveAudio('output.wav', [response.audio], { sampleRate: 24000 });
```

### speak 方法支持哪些输入类型？

`speak` 方法的 `input` 参数支持：

- **字符串**：直接传入文本，适合固定文本的流式合成
- **AsyncIterable\<string\>**：流式文本输入，适合 LLM 流式输出转语音
- **OpenAI Stream**：直接接收 OpenAI SDK 的流式输出

### Opus 格式输出如何保存和播放？

CosyVoice 的 Opus 格式输出使用 OGG 容器封装，保存时使用 `.ogg` 扩展名：

```typescript
const tts = createTTS({
  provider: 'qwen',
  apiKey,
  format: 'opus',
});

const response = await tts.synthesize({ text: '你好世界' });
// 保存为 .ogg 文件
writeFileSync('output.ogg', response.audio);

// 播放
// ffplay -autoexit output.ogg
```

### instructions 参数支持哪些内容？

Realtime 模型的 `instructions` 参数支持自然语言描述，可以控制：

- 情感：如 "请用温柔的语气说话"、"用激动的语气播报"
- 语速：如 "请慢速朗读"
- 风格：如 "像新闻主播一样播报"

<Callout type="info">
`instructions` 参数仅在 `qwen3-tts-instruct-flash-realtime` 模型上生效。CosyVoice 模型请使用 `instruction` 参数。
</Callout>
