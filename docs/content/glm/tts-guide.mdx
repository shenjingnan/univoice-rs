title: 智谱 GLM TTS 使用指南
---

本文介绍如何使用 univoice SDK 中的智谱 AI GLM TTS（文字转语音）功能。

## 概述

GlmTTS 基于智谱 AI GLM TTS HTTP REST API，使用 **HTTP POST** 实现语音合成，支持两种合成模式：

| 模式 | 方法 | 说明 | 适用场景 |
|------|------|------|----------|
| 非流式 | `synthesize` | 一次性返回完整音频 | 文件合成、离线处理 |
| 流式 | `speak` + `stream: true` | 分块接收音频数据 | 实时播报、LLM 流式输出转语音 |

<Callout type="warning">
GlmTTS 的流式模式会先收集完所有文本再发送请求（非真正边发边收），但接口使用方式与其他提供商一致，便于统一调用。
</Callout>

## 创建 TTS 实例

univoice 提供两种创建 GlmTTS 实例的方式：

<Tabs items={['工厂函数（推荐）', '直接实例化']}>
<Tab>

使用 `createTTS` 工厂函数，需先导入 `univoice/tts/providers` 注册提供商：

```typescript
import 'univoice/tts/providers';
import { createTTS } from 'univoice/tts';

const tts = createTTS({
  provider: 'glm',
  apiKey: process.env.GLM_API_KEY,
  voice: 'tongtong',
  format: 'wav',
});
```

</Tab>
<Tab>

直接导入 `GlmTTS` 类并实例化，无需注册 provider：

```typescript
import { GlmTTS } from 'univoice/tts/providers';

const tts = new GlmTTS({
  apiKey: process.env.GLM_API_KEY,
  voice: 'tongtong',
  format: 'wav',
});
```

</Tab>
</Tabs>

<Callout type="info">
两种方式创建的实例拥有完全一致的 API，推荐使用工厂函数，便于切换不同提供商。
</Callout>

## 非流式合成

最简单的使用方式 —— 传入文本，一次性获取完整音频：

```typescript
import 'univoice/tts/providers';
import { createTTS } from 'univoice/tts';

const tts = createTTS({
  provider: 'glm',
  apiKey: process.env.GLM_API_KEY,
  voice: 'tongtong',
  format: 'wav',
});

// 传入文本，返回完整音频
const response = await tts.synthesize({
  text: '欢迎来到杭州！我是您的智能导游。',
});

console.log('音频大小:', response.audio.length, 'bytes');
console.log('音频格式:', response.format);
```

### TTSResponse 结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `audio` | `Buffer \| Uint8Array` | 音频数据 |
| `format` | `string` | 音频格式（如 `wav`、`pcm`） |
| `duration` | `number?` | 音频时长 |

## 流式合成

流式合成适合实时场景，可接收 LLM 流式文本输出：

```typescript
import 'univoice/tts/providers';
import { createTTS } from 'univoice/tts';

const tts = createTTS({
  provider: 'glm',
  apiKey: process.env.GLM_API_KEY,
  voice: 'tongtong',
  format: 'pcm',
});

// 传入字符串，流式接收音频块
for await (const { audioChunk } of tts.speak('欢迎来到杭州！', { stream: true })) {
  console.log('收到音频块:', audioChunk.length, 'bytes');
}
```

也支持传入文本流（如 LLM 流式输出）：

```typescript
import 'univoice/tts/providers';
import { createTTS } from 'univoice/tts';

const tts = createTTS({
  provider: 'glm',
  apiKey: process.env.GLM_API_KEY,
  voice: 'tongtong',
  format: 'pcm',
});

// 模拟 LLM 流式输出
async function* textStream() {
  yield '欢迎来到杭州！';
  yield '我是您的智能导游。';
}

// 流式文本输入 → 流式音频输出
for await (const { audioChunk } of tts.speak(textStream(), { stream: true })) {
  console.log('收到音频块:', audioChunk.length, 'bytes');
}
```

### TTSStreamChunk 结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `audioChunk` | `Uint8Array` | 音频数据块 |

<Callout type="warning">
流式模式仅支持 PCM 格式。即使在创建实例时指定了 `format: 'wav'`，流式输出仍会使用 PCM 格式。如需 WAV 文件，请使用 `saveAudio` 工具函数自动添加 WAV 头。
</Callout>

## 音频格式支持

GlmTTS 支持以下音频格式，通过 `format` 参数指定：

<Tabs items={['WAV', 'PCM']}>
<Tab>

WAV 格式包含文件头，可直接播放，适合文件保存场景：

```typescript
const tts = createTTS({
  provider: 'glm',
  apiKey,
  format: 'wav',
});

const response = await tts.synthesize({ text: '你好世界' });
// response.audio 为 WAV 数据，可直接保存为 .wav 文件
```

</Tab>
<Tab>

PCM 是原始音频数据，无压缩、无容器封装，仅支持非流式和流式模式：

```typescript
const tts = createTTS({
  provider: 'glm',
  apiKey,
  format: 'pcm',
});

const response = await tts.synthesize({ text: '你好世界' });
// response.audio 为 PCM 原始数据
```

<Callout type="info">
PCM 格式不包含文件头信息，播放时需要指定参数。默认配置为 24000Hz、16bit、单声道。
</Callout>

</Tab>
</Tabs>

<Callout type="info">
除 `wav` 和 `pcm` 外的其他格式会自动回退为 WAV 格式。
</Callout>

## 完整配置参考

### BaseTTSOptions（GlmTTS 使用通用配置）

#### 认证配置

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `apiKey` | `string` | `''` | 智谱 AI API Key（**必填**） |

#### 合成配置

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `voice` | `string` | `'tongtong'` | 发音人音色 |
| `format` | `'wav' \| 'pcm'` | `'pcm'` | 音频格式（其他格式回退为 wav） |
| `speed` | `number` | `1.0` | 语速倍率 |
| `volume` | `number` | `1.0` | 音量倍率 |
| `pitch` | `number` | `1.0` | 音调倍率 |

#### 通用配置

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `baseUrl` | `string` | `'https://open.bigmodel.cn/api/paas/v4/audio/speech'` | REST API 地址 |
| `model` | `string` | `'glm-tts'` | 模型名称 |
| `language` | `string` | `'zh-CN'` | 语言 |

## 工具函数

### collectAudio

收集音频数据，支持回调：

```typescript
import { collectAudio } from 'univoice/tts';

const response = await tts.synthesize({ text: '你好世界' });
const audio = await collectAudio(response, {
  onComplete: (audio) => console.log('音频大小:', audio.length),
});
```

### saveTTSResponse

将 TTS 响应保存到文件：

```typescript
import { saveTTSResponse } from 'univoice/tts';

const response = await tts.synthesize({ text: '你好世界' });

// 自动生成文件名
const filepath = await saveTTSResponse(response);

// 指定文件名和目录
await saveTTSResponse(response, {
  filename: 'output.wav',
  directory: './audio',
});
```

### saveAudio

将音频块保存到文件，支持自动添加 WAV 头：

```typescript
import { saveAudio } from 'univoice/tts';

// 保存流式音频块（PCM → WAV）
const stream = tts.speak('你好世界', { stream: true });
await saveAudio('output.wav', stream, { sampleRate: 24000 });
```

### playAudio

播放音频（需要系统安装播放器，macOS 默认使用 `afplay`）：

```typescript
import { playAudio } from 'univoice/tts';

const response = await tts.synthesize({ text: '你好世界' });
await playAudio(response);

// 指定播放器
await playAudio(response, { player: 'ffplay' });
```

### teeAudio

同时保存和播放音频：

```typescript
import { teeAudio } from 'univoice/tts';

const response = await tts.synthesize({ text: '你好世界' });
await teeAudio(response, {
  save: { filename: 'output.wav', directory: './audio' },
  play: { player: 'afplay' },
});
```

## 常见问题

### 非流式和流式模式如何选择？

- **非流式**（`synthesize`）：传入完整文本，一次性获取音频。适合文件合成、离线处理
- **流式**（`speak` + `stream: true`）：分块接收音频数据。适合实时播报、LLM 流式输出转语音

<Callout type="warning">
注意：GlmTTS 的流式模式会先收集完所有文本再发送请求，并非真正的边发边收。但接口使用方式与其他提供商一致，方便统一调用和后续迁移。
</Callout>

### 如何获取智谱 AI API Key？

1. 前往 [智谱 AI 开放平台](https://open.bigmodel.cn/)
2. 注册并登录账号
3. 在控制台中创建 API Key

### PCM 格式音频如何播放？

PCM 是原始音频数据，需要指定参数播放：

```bash
ffplay -autoexit -f s16le -ar 24000 -ac 1 output.pcm
```

或者使用 `saveAudio` 保存为 WAV 格式（自动添加文件头）：

```typescript
import { saveAudio } from 'univoice/tts';

const response = await tts.synthesize({ text: '你好世界' });
// 保存为 WAV 文件，自动添加文件头
await saveAudio('output.wav', [response.audio], { sampleRate: 24000 });
```

### speak 方法支持哪些输入类型？

`speak` 方法的 `input` 参数支持：

- **字符串**：直接传入文本，适合固定文本的合成
- **AsyncIterable\<string\>**：流式文本输入，适合 LLM 流式输出转语音
- **OpenAI Stream**：直接接收 OpenAI SDK 的流式输出

### 如何选择音色？

GlmTTS 提供 9 种内置音色，默认使用 `tongtong`（童童）。通过 `voice` 参数切换：

```typescript
const tts = createTTS({
  provider: 'glm',
  apiKey,
  voice: 'male', // 使用男声音色
});
```
